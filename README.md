# Ансамбль «Лебёдия» — официальный сайт

Женский вокально-инструментальный ансамбль. Гусли, калимба, джембе и русский фольклор в диалоге с современной музыкой.

---

## ⚡ Самый простой способ опубликовать сайт (через GitHub Desktop)

### Шаг 1. Установите GitHub Desktop
Скачайте с https://desktop.github.com и установите. Это бесплатная программа от GitHub, которая упрощает работу с кодом — никакой командной строки.

### Шаг 2. Зарегистрируйтесь на GitHub
Если ещё нет аккаунта — зайдите на https://github.com и создайте (бесплатно).

### Шаг 3. Войдите в GitHub Desktop
Откройте GitHub Desktop, нажмите **Sign in to GitHub.com** и введите свой логин/пароль.

### Шаг 4. Создайте репозиторий из папки
1. Распакуйте архив `lebedia-site.zip` в любое место (например, в Документы)
2. В GitHub Desktop: **File** → **New repository...**
3. Заполните:
   - **Name:** `lebedia-site`
   - **Local path:** нажмите **Choose...** и выберите распакованную папку `lebedia`
   - **Initialize this repository with...** — оставьте пустым
4. Нажмите **Create repository**

### Шаг 5. Сделайте первый коммит
1. В GitHub Desktop слева в поле **Summary** напишите: «Первая версия сайта»
2. Нажмите кнопку **Commit to main**
3. Над кнопкой появится надпись **Publish repository** — нажмите её
4. В появившемся окне оставьте галочку **Keep this code private** снятой (или поставьте, если хотите закрытый репозиторий) → нажмите **Publish repository**

### Шаг 6. Подключите Vercel
1. Зайдите на https://vercel.com
2. Нажмите **Sign Up** → **Continue with GitHub**
3. Авторизуйте Vercel для доступа к вашим репозиториям
4. Нажмите **Add New...** → **Project**
5. В списке выберите `lebedia-site`
6. Нажмите **Deploy** (никакие настройки менять не нужно — Vercel сам определит Next.js)

### Шаг 7. Готово! 🎉
- Через 1–2 минуты сайт будет доступен по адресу вида `lebedia-site.vercel.app`
- Vercel пришлёт письмо со ссылкой
- Каждый раз, когда вы будете менять код в GitHub Desktop и нажимать **Push origin**, Vercel автоматически обновит сайт

## Альтернатива: через командную строку (для тех, кто умеет)

Если у вас установлен git и вы залогинены на GitHub:

```bash
# Распакуйте архив и перейдите в папку
cd lebedia

# Локальный git-репозиторий уже создан и первый коммит уже сделан
# Осталось только подключить удалённый репозиторий и запушить

# Создайте пустой репозиторий на GitHub (без README/gitignore)
# Скопируйте его URL, например: https://github.com/USERNAME/lebedia-site.git

git remote add origin https://github.com/USERNAME/lebedia-site.git
git push -u origin main
```

После этого подключите Vercel как в шаге 6 выше.

## Подключение своего домена (опционально)

Если хотите адрес `lebedia.ru` вместо `lebedia-site.vercel.app`:
1. Купите домен на reg.ru или nic.ru (≈ 200–500 руб/год)
2. В Vercel откройте проект → **Settings** → **Domains**
3. Добавьте ваш домен
4. Vercel покажет, какие DNS-записи поменять у регистратора домена (обычно 2 записи: A и CNAME)
5. Зайдите в личный кабинет reg.ru/nic.ru и пропишите эти записи
6. Через 10–30 минут домен начнёт работать (иногда до 24 часов)

## Что внутри проекта

**Технологии:**
- Next.js 16 + TypeScript
- Tailwind CSS 4 + shadcn/ui
- Prisma ORM (база данных — на сайте пока не используется, оставлена на будущее)

**Структура:**
- `src/app/` — страницы (главная `page.tsx`, layout)
- `src/components/site/` — компоненты сайта ансамбля
  - `Variant1Lebedia.tsx` — основной (и единственный используемый) вариант дизайна
  - `Variant2Hyperborea.tsx`, `Variant3Skazka.tsx` — альтернативные концепции (не используются)
  - `data.ts` — все тексты ансамбля (название, биография, репертуар, достижения и т.д.)
  - `Reveal.tsx` — компонент анимации появления
- `src/components/ui/` — библиотека shadcn/ui
- `public/` — логотипы и фотографии
- `prisma/schema.prisma` — схема базы данных

**Чтобы изменить тексты** — отредактируйте `src/components/site/data.ts` (можно прямо в GitHub через веб-редактор, нажав на файл и кнопку ✏️). Сайт обновится автоматически через 1–2 минуты.

## Поддержка

Если что-то не работает — проверьте:
1. Что в GitHub загрузились все файлы (особенно `package.json` и папка `src/`)
2. Что в Vercel выбран правильный репозиторий
3. Логи сборки в Vercel: вкладка **Deployments** → нажмите на последний деплой → **Build Logs**

Если Vercel выдаёт ошибку сборки — скопируйте текст ошибки и спросите, поможем разобраться.
